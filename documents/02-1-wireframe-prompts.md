# ワイヤーフレーム作成用プロンプト集

## プロンプトの使い方

以下のプロンプトをAI画像生成ツール（Claude、ChatGPT、V0、Figma AI等）に入力してワイヤーフレームを生成します。
各画面ごとに分けて作成することを推奨します。

---

## 1. 記事一覧画面（パブリック - 閲覧者向け）

```
MyBlog-AWSプロジェクトの記事一覧画面のワイヤーフレームを作成してください。

【画面の目的】
- 公開されている記事の一覧を表示し、閲覧者が興味のある記事を見つけられるようにする

【必要な要素】
1. ヘッダーエリア
   - サイトロゴ/タイトル「MyBlog」
   - ナビゲーションメニュー（Home, About, Contact等）
   
2. メインコンテンツエリア
   - 記事カード（グリッドレイアウト、2-3列）
     * サムネイル画像
     * 記事タイトル
     * 投稿日時
     * カテゴリータグ
     * 記事の要約（1-2行）
   
3. サイドバー（オプション）
   - カテゴリー一覧
   - 最新記事
   - 検索ボックス
   
4. フッターエリア
   - コピーライト
   - SNSリンク
   - プライバシーポリシー等

【デザイン指針】
- シンプルで清潔感のあるデザイン
- モダンなブログUIを参考
- レスポンシブデザイン対応を考慮
- 画像とテキストのバランスを重視
```

## 2. 記事詳細画面（パブリック - 閲覧者向け）

```
MyBlog-AWSプロジェクトの記事詳細画面のワイヤーフレームを作成してください。

【画面の目的】
- 個別記事の全文を表示し、画像・動画を含むリッチコンテンツを提供する

【必要な要素】
1. ヘッダーエリア
   - 記事一覧画面と同じ構成

2. 記事メタ情報エリア
   - 記事タイトル（大きく目立つ）
   - 投稿日時
   - カテゴリータグ
   - 著者情報（アイコン + 名前）

3. 記事本文エリア（ブロック構造）
   - テキストブロック（段落、見出し）
   - 画像ブロック（キャプション付き）
   - 動画ブロック（YouTube埋め込み or S3動画プレーヤー）
   - 各ブロックは視覚的に区別可能

4. アクションエリア
   - SNSシェアボタン（Twitter, Facebook, LinkedIn等）
   - 「記事一覧に戻る」ボタン

5. 関連記事エリア（オプション）
   - 同じカテゴリーの記事を3-4件表示

6. フッターエリア
   - 記事一覧画面と同じ構成

【デザイン指針】
- 読みやすさを最優先（適切な行間、文字サイズ）
- 画像は横幅いっぱいに表示可能
- ブロック間に適度な余白
- 長文でも疲れないデザイン
```

## 3. 管理画面 - 記事一覧（管理者向け）

```
MyBlog-AWSプロジェクトの管理画面（記事一覧）のワイヤーフレームを作成してください。

【画面の目的】
- 管理者が全ての記事（下書き・公開・アーカイブ）を一覧で確認し、操作できるようにする

【必要な要素】
1. ヘッダーエリア
   - 管理画面タイトル「MyBlog - Admin」
   - ログアウトボタン
   - ユーザー名表示

2. サイドナビゲーション
   - ダッシュボード
   - 記事一覧（現在表示中）
   - 新規記事作成
   - メディアライブラリ
   - 設定

3. メインコンテンツエリア
   - ページタイトル「記事一覧」
   - アクションボタン
     * 「新規記事作成」ボタン（目立つ配置）
   
   - フィルター/検索バー
     * ステータスフィルター（全て/下書き/公開/アーカイブ）
     * カテゴリーフィルター
     * 検索ボックス
   
   - 記事テーブル（リスト形式）
     * サムネイル（小）
     * タイトル
     * ステータスバッジ（DRAFT/PUBLISHED/ARCHIVE）
     * カテゴリー
     * 作成日時
     * 更新日時
     * アクション（編集/削除ボタン）

   - ページネーション

【デザイン指針】
- ダッシュボードUIを参考（Material Design, Ant Design等）
- 操作性を重視
- ステータスは色分けで視認性向上
- テーブルはソート可能
```

## 4. 管理画面 - 記事編集/作成（管理者向け）

```
MyBlog-AWSプロジェクトの管理画面（記事編集/作成）のワイヤーフレームを作成してください。

【画面の目的】
- ブロック構造のコンテンツを作成・編集できるエディタ画面

【必要な要素】
1. ヘッダーエリア
   - 「記事を編集」または「新規記事作成」タイトル
   - アクションボタン
     * 「下書き保存」ボタン
     * 「公開」ボタン
     * 「プレビュー」ボタン
     * 「戻る」ボタン

2. メインエディタエリア（左側 60-70%）
   - 記事タイトル入力欄（大きな入力フィールド）
   
   - コンテンツブロックエリア
     * ブロック追加ボタン「+ ブロックを追加」
     * 各ブロックの表示
       - テキストブロック：リッチテキストエディタ
       - 画像ブロック：画像プレビュー + アップロードボタン + キャプション
       - 動画ブロック：動画URL入力 or アップロードボタン
     * 各ブロックにドラッグハンドル（並び替え用）
     * 各ブロックに削除ボタン

3. サイドバー（右側 30-40%）
   - 記事設定セクション
     * ステータス選択（DRAFT/PUBLISHED/ARCHIVE）
     * カテゴリー選択/作成
     * タグ入力
     * サムネイル画像設定
     * 公開日時設定
     * URLスラッグ編集

   - SEO設定セクション（オプション）
     * メタディスクリプション
     * OGP画像

【デザイン指針】
- Notion、WordPress Gutenberg、Medium等のエディタを参考
- ブロックの追加・削除・並び替えが直感的
- ドラッグ&ドロップ対応を示唆
- WYSIWYG（見たまま編集）を意識
```

## 5. ログイン画面（管理者向け）

```
MyBlog-AWSプロジェクトのログイン画面のワイヤーフレームを作成してください。

【画面の目的】
- Amazon Cognitoを利用した管理者認証

【必要な要素】
1. 中央配置のログインカード
   - サイトロゴ/タイトル「MyBlog Admin」
   - メールアドレス入力フィールド
   - パスワード入力フィールド
   - 「ログイン」ボタン（目立つ）
   - 「パスワードを忘れた場合」リンク

2. 背景
   - シンプルで邪魔にならないデザイン
   - グラデーションまたは単色

【デザイン指針】
- シンプルで信頼感のあるデザイン
- モバイル対応
- セキュリティを感じさせるUI
```

## 6. モバイル版の考慮

各画面のモバイル版も検討する場合は、以下を追加：

```
上記の[画面名]について、モバイル版（スマートフォン 375px幅）のワイヤーフレームも作成してください。

【モバイル特有の要素】
- ハンバーガーメニュー（ナビゲーション）
- 1カラムレイアウト
- タッチ操作に適したボタンサイズ
- スワイプジェスチャーの考慮
```

## プロンプト使用のヒント

1. **段階的に作成**: 全画面を一度に依頼せず、1画面ずつ作成
2. **反復改善**: 生成されたワイヤーフレームを見て、不足要素を追加指示
3. **ツール選択**: 
   - 低忠実度（Lo-Fi）: 手書き風のシンプルなワイヤーフレーム
   - 高忠実度（Hi-Fi）: よりデザインに近いモックアップ
4. **参考サイト指定**: 特定のサイトのレイアウトを参考にしたい場合は明示
5. **フォーマット指定**: HTML/CSS、Figmaファイル、画像等、出力形式を指定
